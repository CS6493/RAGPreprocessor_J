RAGPreprocessor
===============

Context / Purpose
-----------------
This repository contains a preprocessing script used to prepare a RAG (Retrieval-Augmented Generation)
corpus from the Hugging Face HotpotQA dataset. It:

1) Loads HotpotQA (default: hotpotqa/hotpot_qa, split: distractor/train subset)
2) Cleans extracted documents
3) Chunks documents using multiple (chunk_size, overlap) configurations (ablation study)
4) Saves chunked corpora to JSON
5) Builds sparse retrieval indices (BM25) for EACH chunk configuration
6) Builds a dense retrieval index (Contriever embeddings + FAISS) for ONE default chunk configuration


Repository Structure
--------------------

Scripts
  - preprocess.py
      Main script. Defines class `RAGPreprocessor` and (when run directly) executes the full pipeline:
      read_data -> clean_data -> chunk_data -> save_all_chunks -> build_all_bm25 -> build_dense_index

Data / Outputs (generated by running preprocess.py)
  Chunk JSONs (4 configurations)
    - chunks_size_256_overlap_25.json
    - chunks_size_256_overlap_50.json
    - chunks_size_512_overlap_25.json
    - chunks_size_512_overlap_50.json

  BM25 Sparse Indexes (4 configurations; saved as pickle)
    - bm25_size_256_overlap_25.pkl
    - bm25_size_256_overlap_50.pkl
    - bm25_size_512_overlap_25.pkl
    - bm25_size_512_overlap_50.pkl

  Contriever Dense Index (default chunk configuration only)
    - contriever_default.index
        FAISS index storing normalized dense embeddings (cosine similarity via inner product).
    - contriever_default_metadata.json
        Maps FAISS row -> original chunk text (and chunk_id) so you can recover the text after retrieval.

What the Script Actually Does (Pipeline Details)
------------------------------------------------

Step 1) read_data(num_samples=50)
  - Downloads HotpotQA from Hugging Face via `datasets.load_dataset`.
  - Extracts HotpotQA "context" into a single text document per record:
      Title: <title>
      <sentences...>
      (repeated for each title)
  - Default is 50 samples for faster experimentation. Use num_samples=None for the full train set.

Note: The script sets:
  HF_ENDPOINT=https://hf-mirror.com
This is intended to help with Hugging Face connectivity in some regions.

Step 2) clean_data(min_length=20)
  - Normalizes whitespace/newlines and removes very short documents.

Step 3) chunk_data(configs=None, default_key=None)
  - Uses LangChain's RecursiveCharacterTextSplitter.
  - Default ablation configs (4):
      size=256 overlap=25
      size=256 overlap=50
      size=512 overlap=25
      size=512 overlap=50
  - Adds metadata to each chunk: doc_id, dataset, split, chunk_size, chunk_overlap, chunk_id.
  - The default chunks used for dense indexing are:
      default_key = "size_512_overlap_50"   (unless you override it)

Step 4) save_all_chunks(output_dir="output")
  - Writes JSON files, one per chunk configuration.
  - Each JSON record includes: chunk_id, doc_id, dataset info, chunk params, and the chunk text.
  - In the provided main block, output_dir is set to "" (empty string), which saves files in the repo root.

Step 5a) build_all_bm25(output_dir="output")
  - Builds a BM25Okapi index per chunk configuration.
  - Uses basic tokenization: lowercasing + split on spaces.
  - Saves a pickle containing:
      - bm25 model
      - tokenized_corpus
      - chunk_records (page_content + metadata)

Step 5b) build_dense_index(model_name="facebook/contriever-msmarco", output_dir="output")
  - Loads a SentenceTransformer model and encodes the default chunk set only.
  - Normalizes vectors and builds a FAISS IndexFlatIP for cosine similarity search.
  - Saves:
      - contriever_default.index
      - contriever_default_metadata.json (list of {chunk_id, text})
  - If dependencies are missing, it prints install instructions and skips this step.


How to Run (Windows / PowerShell)
---------------------------------

1) Create and activate a virtual environment (recommended)
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1

2) Install dependencies
   Required for dataset + cleaning + chunking + BM25:
     pip install datasets langchain-text-splitters rank-bm25

   Required only for dense retrieval indexing (Contriever + FAISS):
     pip install sentence-transformers faiss-cpu numpy

3) Run preprocessing
   python preprocess.py

After it finishes, you should see the JSON chunk files and BM25 .pkl files in the output directory.
In the current script, outputs are saved in the repository root because output_dir="" is used.


Common Customizations
---------------------

- Change dataset or split:
    preprocessor = RAGPreprocessor(dataset_name="hotpotqa/hotpot_qa", split_name="distractor")

- Use more data:
    preprocessor.read_data(num_samples=None)

- Change chunking configs:
    preprocessor.chunk_data(configs=[{"size": 400, "overlap": 50}], default_key="size_400_overlap_50")

- Save outputs into a folder:
    preprocessor.save_all_chunks(output_dir="output")
    preprocessor.build_all_bm25(output_dir="output")
    preprocessor.build_dense_index(output_dir="output")


Notes / Caveats
---------------
- Dense indexing can be slow and memory-heavy depending on how many chunks you generate.
- FAISS + SentenceTransformers install may vary by environment; if you only need BM25, you can skip them.
- The HotpotQA loader is called with:
    verification_mode="no_checks", trust_remote_code=True
  which can speed up loading but has security implications (trusting remote code).
